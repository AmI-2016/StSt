Project Specifications

Summary:

Purpose and Scope, Definitions, Requirements and Technical Specifications.

Purpose and Scope

The aim of the project is to improve the working terminal of the user thanks to devices which correct wrong posture of the body and give the possibility to regulate correctly room’s brightness.

The system is composed by the following parts:
  • A wearable device with a set of accelerometers that get informations about posture and a vibrator. The latter is activated by the system to alert the user of a wrong posture.
  • A light-sensor that sense the environment brightness.
  • A PC used as data control unit of the entire structure. It is used by the user to start the system and it's able to show the data collected of the current session.
  • A website that shows all the statistics of the previous sessions and suggests user how to improve his working terminal.

Definitions

Glossary

Location
The personal working terminal: at home, in the office or in any room that can be personalized.
Wearable device
It is a T-shirt worn by the user during the study session. Some accelerometers and a vibrating device are placed on it.
“StSt” device
It is the union of the wearable device and the light sensor.
Website
Website of “Study Station”: used to show all the results to the user.
Software
Program that controls all data received to the “StSt” device. It is independent to the website but they communicate between each other.
User Interface
Graphic interface of the software that allows user to interact with hardware parts of “StSt” device.

Actors

Target user
User who is approaching to study or work on their working terminal using “Study Station”.
Administrator
User that accesses software or website to execute the following actions: log in, start the study session, close the study session, get results of the data, log out.

Functional Requirements

  1. Notifications - A vibration warns user to change posture every time it is wrong; a vibrator placed on the “StSt” device must generate the vibration. (1)
  2. Controller - The software must give the possibility to start and stop “StSt” device computation. (1)
  3. Controller - Data are visible in real-time on the software. (2)
  4. Software - The software must have a user interface. (2)
  5. Notifications - Sound signals and displayed messages which suggest user to modify the environment brightness are produced by the software. (2)
  6. Statistics - The software must provide statistics about data received from sensors about shoulders inclination and environment brightness of the current study session. (3)
  7. Statistics - The website must show a report and a chronology related to all statistics collected during the previous study sessions. (3)
  8. Log - The user must be able to log in/out the website (the website must be able to collect and retrieve data from more than one user). (4)

Non Functional Requirements

  1. Portability - The system must be compatible with Windows.
  2. Usability - The system supports English language.
  3. Reliability - “StSt” device needs to be connected to PC through USB and so it is necessary that software must run during all session time.
  4. Implementation -“StSt” device must be worn correctly by the user to work in the right way: sensors must be fixed, so it is necessary to implement a close-fitting wearable device.
