Purpose and Scope

The aim of the project is to improve the working terminal of the user thanks to devices which correct wrong posture of the body and give the possibility to regulate correctly room’s brightness.

The system is composed by the following parts:

  • A wearable device with a set of tilt sensors that give informations about posture.
  • A light-sensor that controls the environment brightness and computes some operations to fix it in order to avoid sight tiredness.
  • A software used as data control unit of the entire structure.
  • A website that shows all the data and suggests user how to improve his working terminal (this function can be realized also by the software).

    • Definitions

    Glossary

  • Location: the personal working terminal: at home, in the office or in a room that can be personalized.
  • Website: Website of “StudyStation”: used to show all the results.
  • “StSt” device: It is the union of the wearable device and the light sensor.
  • Wearable device: It is a T-shirt worn by the user during the study session; the tilt sensors and the vibrating device are placed on it.
  • Software: Program that controls all data received to the “StSt” device. It is independent to the website but they communicate between each other.
  • User Interface: Graphic interface of the software that allows user to interact with hardware parts of “StSt” device.

    • Actors

  • Target user: user who is approaching to study or work on their working terminal using “StudyStation”.
  • Administrator: user that accesses software or website to execute the following actions: log in, start the study session, close the study session, get results of the data, log out.

    • Functional Requirements

    LOG
  • The user must be able to log in/out the website. (4)
    • STATISTICS
  • The software must provide statistics about data received from sensors about shoulders inclination and environment brightness. (2)
  • The website must show a report and a chronology related to all statistics collected during the study session. (3)
    • NOTIFICATIONS
  • A vibration warns user to change posture every time it is wrong; a vibrator placed on the “StSt” device must generate the vibration. (1)
  • Sound signals and displayed messages which suggest user to modify the environment brightness are produced by the software. (2)
    • CONTROLLER
  • The software must give the possibility to start and stop “StSt” device computation. (1)
  • Data are visible in real-time on the software. (2)
    • SOFTWARE
  • The software must have an user interface. (2)

    • Non Functional Requirements

  • The system must be compatible with Windows. (Area: Portability)
  • The system supports English language. (Area: Usability)
  • “StSt” device needs to be connected to PC through USB and so it is necessary that software must run during all session time. (Area: Reliability)
  • “StSt” device must be worn correctly by the user to work in the right way: sensors must be fixed, so it is necessary to implement a close-fitting wearable device. (Area: Implementation)
